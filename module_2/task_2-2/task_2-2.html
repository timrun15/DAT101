  2. How do parentheses affect the arithmetic outcome of an expression?

    3. Parentheses in an arithmetic expression dictate the order of operations, ensuring that the operations within the parentheses are performed first. This can change the outcome of the expression by overriding the default precedence rules (PEMDAS/BODMAS), where operations like multiplication and division typically take precedence over addition and subtraction.

  3. How do we limit floating-point numbers to a given number of decimals?

    4. To limit floating-point numbers to a specific number of decimal places in JavaScript, you can use the .toFixed() method. For example, number.toFixed(2) will round the number to two decimal places. This method returns the result as a string.

  4. How can we compare two floating-point values to see if they are equal (is 12.199999 the same as 12.2)?

    5. To compare two floating-point values for equality, you should check if their difference is within a small tolerance or epsilon value, due to potential precision issues. For example, abs(a - b) < epsilon, where epsilon is a small number like 1e-9, can be used to determine if a and b are effectively equal.

  5. What do comparison operators return, and how can we check if they are FALSE or TRUE?

    6. Comparison operators return a boolean value: true or false. You can check this by evaluating the expression directly in a condition, like an if statement.

  6. How does the Math.random() function work, and what can it be used for?

    7. The Math.random() function generates a floating-point number between 0 (inclusive) and 1 (exclusive). It can be used for tasks like generating random numbers, selecting random items from an array, or creating randomized behavior in games and simulations. To get a random number in a different range, you can scale and shift the result, such as Math.random() * 10 for a number between 0 and 10.
